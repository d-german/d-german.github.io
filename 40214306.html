Hyland Clinician Window : Pillars of OOP - Encapsulation/Abstraction

Created by Damon German, last modified on Aug 17, 2021
  • Encapsulation: The idea of bundling related data and behavior within one entity.
  • Abstraction: Information Hiding
    • Don't care or need to know about the internals of a doorknob to access it through its public interface. Turn().
  • Guidelines
    • All data members should be private 
    • Methods needed only by other methods should be private
    • Only methods needed to be called by other classes are public or protected
    • The public methods should not be too implementation specific.
      • Save() vs  SaveToDatabase()
    •  Public method signatures should be as stable as possible. (Any dependencies will need to be updated.)
