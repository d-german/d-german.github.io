Hyland Clinician Window : When to use Target-typed new expressions

Created by Drew Hudson on Sep 08, 2022

https://docs.microsoft.com/en-us/dotnet/csharp/language-reference/proposals/csharp-9.0/target-typed-new

We shouldn't be using new() frequently unless it enhances visibility and readability.

This would be a suitable time to use it:
private readonly Dog dog = new Dog();
could be improved to be
private readonly Dog dog = new();

The following example is NOT the time to use it:

 var dictionary = new Dictionary<object, RepositoryResponse<PatientContents>>
            {
                [categoryId1] = new()
                {
                    ResponseObject = new()
                    {
                        Items = new()
                        {
                            new()
                            {
                                Type = type1
                            },
                            new()
                            {
                                Type = type2,
                                Restricted = true
                            }
                        }
                    }
                },
                [categoryId2] = new()
                {
                    ResponseObject = new()
                    {
                        Items = new()
                        {
                            new()
                            {
                                Type = type3,
                                Restricted = false
                            }
                        }
                    }
                },
                [categoryId3] = new()
                {
                    ResponseObject = new()
                    {
                        Items = new()
                    }
                }
            };

In the above example, the developer now must hover over the variable in an IDE with the mouse to see what type it is. A reader without an IDE wouldnâ€™t know.
